# Pixelto UI 极致优化实施计划 (Revised)

本计划严格遵循“分阶段、强审核”的原则。每个阶段完成后，需等待用户确认通过，方可进入下一阶段。

## 阶段一：主题设计与视觉验证 (Theme Design & Verification)

**目标**：确立全站视觉基调，通过独立的“设计系统预览页”验证深浅色模式下的所有核心元素。

### 1.1 主题引擎重构 (Theme Engine)
- [x] **定义语义化变量** (`theme.css`):
    - 基础层：`--bg-page`, `--bg-surface`, `--text-primary` 等。
    - 质感层：`--glass-border`, `--glass-surface`, `--noise-texture`。
    - 能量层：`--glow-primary`, `--glow-accent` (用于 AI 激活状态)。
- [x] **扩展 Tailwind 配置**: 确保所有 CSS 变量都能通过 utility class 调用 (e.g., `bg-surface`, `shadow-glow`).

### 1.2 构建设计系统预览页 (Design System Preview Page)
- [x] **创建新路由**: `/design-system` (仅开发环境或通过特定路径访问)。
- [x] **实现预览模块**:
    - **Typography**: 展示各级标题、正文、代码字体的在深浅色下的表现。
    - **Color Palette**: 展示背景色、主色、功能色阶梯。
    - **Components Lab**:
        - Buttons: 普通按钮 vs "流光"按钮 (AI Trigger)。
        - Inputs: 玻璃态输入框交互 (Focus/Hover)。
        - Cards: 玻璃拟态卡片、悬停发光卡片。
    - **AI Specifics**:
        - 模拟 "Floating Control Bar" (悬浮控制岛) 的静态效果。
        - 模拟 "Generating" 状态的加载动效。

### 🛑 **里程碑 1：设计系统验收**
*   **交付物**: `/design-system` 页面 URL。
*   **动作**: 用户查验深浅色模式下的组件质感。
*   **状态**: 待确认。
